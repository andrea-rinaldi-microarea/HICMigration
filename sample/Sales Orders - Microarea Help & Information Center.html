Questa funzione si trova in: Sales | Sales Orders | Sales Orders

The Customer Order usually is the first document in the Sales Process. It is also possible to load totally or partially a customer quotation; if in the Customer Quotation is present a Contact, the program provide for transforming it into a Customer automatically after the code confirmation.

Articolo come... o (Ctrl+Alt+W) Permette di inserire nel campo articolo il codice presso il cliente impostato nell'anagrafica cliente dell'articolo e il programma effettuerà la ricerca dell'articolo ad esso associato.

Modifica

Head data

The number of the order (Number) is proposed automatically, based on an internal numbering sequence that is not fiscal. The Of field is filled in automatically and can be modified manually clicking on the specific option. The Customer Code is a mandatory field and can be entered manually if the user decide to enter a new customer. If a customer quotation is loaded, this field is filled in automatically.

Modifica

Main Data tab


Contenuto incluso da: RefGuide-M4-SalesMainDataTab

Main Data

In this section, the Payment condition is read from the Customer Master or Contact Master and can be changed manually. Through the Prices do not Include Tax checkbox it is possible to indicate unit values net of VAT or included it. The program proposes all documents with amounts net of VAT with the exception of fiscal receipt and non-collected receipts. This proposition can however be modified by acting in Sales Purchases Document Parameters.

In the Delivery Note there is the Invoice Follows parameter which allows, if set, to carry out the fulfillment of the Delivery Note on the invoice.

Currency

The Currency is read from the Customer Master and can be changed manually.

If the Currency on the document is different from the base currency of the company, the Fixing Date field is enabled and the program, based on the data in the Currency Master, proposes the most recent Fixing Date prior to the date of the document and its fixing value.

It is possible to modify this value by opening the drop-down menu of the Currency field which shows all the Fixings in the Currency Master. You can also freely enter the desired fixing value.

Delivery

This section is present only in the Customer Quotations and Customer Orders documents.

Within this section, in the Customer Quotations, there are the fields Deliver on and Expire On as descriptive data.

In the Sales Orders, there are also the Deliver confirmed on, Deliver not later than and Single Delivery dates are automatically shown on the lines of the document.

Detail

In the Detail section it is possible to enter the items specifying the desired data (unit of measurement, quantity, value, discount etc.).

Using the Line Type you can enter:

  • inventory items (Goods Line Type );
  • services (Service Line Type);
  • inventory items that are not moved and that do not participate in the calculation of the total document (Descriptive Line Type);
  • free text in the description field (Note Line Type);
  • the ends of the summarized documents (Reference Line Type). This line type is always entered on the invoices generated from the Deferred Invoicing procedure and reports the references of Delivery Notes.


The Goods Line Type is not available for the Sales Invoice for Advance document.

The Item code, by right clicking in the related field, can be searched using different criteria (by Code or Description of the Customer or Supplier, by Barcode, etc.). For details see: Item Search.

The Item Description is filled automatically by recalling an item description based on what is in the its Master. It is possible to modify it manually on the document itself. This change does NOT affect the Items Master, but only the document on which it has been reported and, possibly, those derived (example: Order -> Delivery Note -> Invoice).

The Unit of Measure (UoM) can be entered manually or selected in the appropriate drop-down menu, choosing between base units of measurement of the item, general units of measurement and alternative units of measure present in the Items Master. For details see: General UoMs, Items.

The Unit Value and Discount can be entered manually or selected in the appropriate drop-down menu choosing between the values ​​provided in the Price List (Prices Policies/Discounts Sales Process). For details see: Price/discount management policies Sales Cycle.

The VAT Code is completed on the basis of what is in the Items Master or, in its absence, the default VAT codes. For details see: VAT Codes, Default Tax Codes.

Depending on the document there are some parameters such as:
  • No Print - the line is not printed (in the printouts set for this check);
  • Not Postable in Inventory - the row is not posted in inventory;

Queste funzionalità sono disponibili solo con: Variant Module
Whenever an Item managed by Variant Type is offered, ordered and sold, the Variant Code must always be indicated in the Sale Document, in addition to the quantity.

RowView

By accessing the Rowview are visible, in addition to all the fields on the grid, other fields. It is however possible to customize the grid and extract any fields using the Customize button (Ctrl-F8).

Main Data

For each line it is possible to indicate the Sales Type:
  • Normal for the common Sale;
  • Taxable Free Sample to grant the customer a gift equal to the taxable value of the item;
  • Total Free Sample to grant the customer a gift equal to the taxable value of the item plus the amount of VAT;
  • Promotion as the normal sale, useful for personalized statistics;
  • Discount on Goods to grant a 100% discount on the line (no tax is therefore calculated).


If the unit of measurement shown in the line provides for the use of the coefficients, the section relating to the additional quantities (Additional Quantities) is enabled and the relative values can be entered. For example, in the case of the Cubic Meters, it is possible to use the coefficients Length, Width and Height and by inserting the relative values in the document and the cubage is calculated automatically.

The Pack, value to be entered manually, allows the printing of the document, through custom fills, ordered by Pack value, to make it easier to check the correspondence between goods indicated on the document and packs (pallets) physically present . In the subsection relating to the Totals there are information concerning the Tax Summary and Shipping Totals in which the Gross Weight and Net Weight fields preserve the details regarding the weight of the goods or handled services.

Other Data

The Department, descriptive data, is proposed equal to that inserted in Item Master.

In the External References the Document Number and the Line fields are free; they can used to enter manually the references to an external document and its line number.

If an order is processed with this document, the number (Order Number) and the (Position) position is displayed and it is possible to force the closure (Close Order Position): receiving a smaller quantity than the ordered one, it is possible to consider the line completely fulfilled (this option is not present in Quotations and Order).

Administrative Data

The Offset is proposed according to the following proposition list:
- Customer Item Master Data;
- Commodity Sales Category Master Data;
- Items Master Data;
- Customer Master Data;
- Offset and Default Codes Master Data.


Queste funzionalità sono disponibili solo con: Intrastat Module.
In the Intrastat section, the data for the generation of the Intrastat entry (from Invoice or Credit Note), required to complete the Intrastat Declaration, in the case of sale to foreign customers belonging to a member state of the EU, are reported.

Summary tab


Contenuto incluso da: RefGuide-M4-SaleSummaryTab

This section summarizes all the amounts related to the document.

Charges

They are calculated automatically by the program, if foreseen, or the charges related to the following list can be possibly modified/entered manually:

  • Collection - to be valued manually according to what is reported in the document fro the customer;
  • Shipment - based on the settings in the Carrier Master Data or to be evaluated manually on the basis of the document for the customer;
  • Packing - a descriptive type field to be evaluated manually according to what is reported in the document for the customer;
  • Additional - based on Customers by items data and enabling in the Sales Parameters the Customer/item additional expenses calculation option.
  • Stamps - to be valorised on the basis of what has to be issued to the customer.


Totals

In this section the Goods, Services and Free Sample fields contain the sum of all items, based on their type that can be consulted through the rowview .

The percentage of Additional Discount can be entered manually or reported automatically by the Payment Condition (typically the payment by cash may include a percentage discount). If necessary, you can manually enter the Discount Amount directly using the Lightning/Pencil button.

Then there are a series of fields that show the details of the amounts of the document; if necessary, you can manually enter an amount of Allowances that will decrease the Total to Be Paid.

Tax Summary

All data relating to VAT are shown in the grid (Tax Summary). At the same VAT number, all amounts of the document are added together and the tax is calculated on this value. Therefore, there are as many lines as VAT codes included in the document, regardless of the number of items.

In Germany localization the Tax Summary has different features. For more detail see Logistics Area

Administrative Data tab


Contenuto incluso da: RefGuide-M4-SaleAdministrativeDataTab

Accounting

The Accounting Template (field not present in the Customer Quotation) is used to account for the Invoice or Credit Note.

Contenuto incluso da: RefGuide-M4-Accounting-Common-TaxCommunication

Codice raggruppamento: è possibile indicare un codice per stabilire quali documenti si riferiscono ad uno stesso contratto. E' possibile indicare anche un codice non censito.

Indica come deve essere considerata l'Operazione: stabilisce se l'operazione deve essere inclusa o esclusa dalla Comunicazione. Il valore predefinito è Automatica, ossia la procedura di generazione della Comunicazione Polivalente stabilisce se il documento debba essere incluso o escluso sulla base dell'importo e del collegamento ad altri documenti.
Se si imposta Esclusa, il documento viene sempre escluso dalla comunicazione.
Se si imposta Inclusa, il documento viene sempre incluso nella comunicazione, anche se non ne avesse i requisiti.

Through the Advances sub-section, not present in the Customer Quotations and in the Customer Orders, it is possible to enter an Amount for payment in advance. To ensure that this is deducted from the total of the document and accounted (exclusively by the invoice, concurrently with the accounting of the same), the Paid concurrently parameter must be set: in this case, it is also possible to specify the Offset on which it is intended to account the deposit itself. In the event that the Paid concurrently parameter has not been set, the entered amount has to be considered purely descriptive.

Through the Link Already Posted Advance check-box (active only in the Invoices) it is possible to deduct from the amount of the document an account previously recorded in the Ledger. For example, by entering a posting of a Ledger with Accounting Template ADVCUST (Advance from Customer), at the saving moment, the program proposes the creation of a Customer Offset characterized by the Advance setting.

The Cost Center and Job fields, present in the sub-section Analytical Accounting, are used to link to Analytical Accounting (if it is present).

In the Recipients section, the Document Recipient field, indicates the location where sending documents of the Customer and, if present in the Customer Master, is automatically proposed. It can be used as information to be printed in the document.

Also the Payments Nominee, if the Payments Addressed to field indicated in the Customer Master is filled, is automatically proposed in the order.

Delivery tab


Contenuto incluso da: RefGuide-M4-Sales-Common-SalesDeliveryTab

Inventory

In the Inventory Reason field it is possible to select the Inventory Reason to be used. By setting this value, when issuing the document, it is possible to generate the inventory entry from any document except from the Sales Order. In the document in which the order is fulfilled, the reason is automatically reported according to which has been set in the order; if it has not been entered, the default reason for the document is used.

The Invoices may or may not include the Inventory Reason. It is however possible to manually enter the reason ven when it is not foreseen. In the case in which, in the Sales Parameters, the Issue of Goods is done with both with Delivery Note and Invoice parameter has been set, the Inventory Reason, indicated in the Sales Default Codes by User in the Invoice not summarizing DN field is proposed in the invoice (Invoice for Quantity and Value).

According to the settings of the Inventory Reason, the Storage Phase 1 (and possibly Storage Phase 2) fields and, if necessary, to obtain a correct identification, even the Specificator of the Phase 1 (and possibly for the Phase 2) are enabled and completed.

By posting an Sales Order, the ordered quantity placed on the Storage entered in Storage Phase 1 is decreased regardless the settings of the inventory reasons; the ordered quantity on the storage will be decreased with the fulfillment of the order in the delivery note or invoice.

Carriers

The First Carrier is proposed by the program according to the settings in the Customer Master Data in the Sales tab. If it is necessary, in the Carriers section (only in the Delivery Note) it is also possible to add a Second Carrier and a Third Carrier.

Shipping

The shipping office may be chosen from those coded in Company, Customer Master Data according to the Delivery to filed.

The Port is proposed based on the settings in the Customer Master Data. By setting the Port as a Free Port no shipping costs has been assigned. It is possible to set a Carriage Fre Threshold value in the Custome Master

In this section, it is possible to define some characteristic of packs that are shipped, for example the Packing (i.e. box) type and its good status Appearance.

The Shipment field indicates who is the responsible of the shipment. It can be automatically proposed according to the Customer master data. When the first carrier is completed, the program automatically proposes the code concerning the shipment by the carrier.

The Transport field indicates how the goods will be shipped. It can be automatically set according to the Customer master data. The Transport Reason indicates the motivation of the shipment.

Transportation Form

Queste funzionalità sono disponibili solo in: Italy.

In Italy the Transportation Form is mandatory if the shipment concerns goods auto-transport activity for third parties on national area. It must be completed and written by the Customer before the transport and must be kept on board the vehicle by the carrier. In this case, the Customer is the person that has ordered the shipping and sign the transportation agreement and he is not not who has ordered the good. By default it is represented by the Company.

In order to manage this form, it necessary to enable the Use Transportation Form parameter.

The Owner Data field is always mandatory and if it is not possible to indicate it, the Customer must give a reason. It is the company or the public legal entity that owns the goods that are the object of the shipment. By default it is the Company.
Loader Data section identifies the company or public legal entity that delivers the goods to the carrier, taking care of arranging the goods on the vehicle used for transport; by default it is equal to the Company.
Declarations field has to be filled in when it is not possible to indicate the figure of the owner.
Loading Place is the place where the goods is loaded in the vehicle used for transport; it is possible to select a company branches.
Unloading Place is the place where the goods is unloaded from the vehicle used for transport; it is possible to select a shipping branch available in the Customer Master. By default it is proposed the place indicated in the Delivery To field in the Shipping tab of the document.
Compilation Place and Compilation Data are free fields to indicate respectively the place where the Transportation Form has been completed and the information of the person that has filled the transportation form.
Compilation Date is set equal to the Document Date.

Item Data

Contenuto incluso da: RefGuide-M4-Documents-Common-ItemDataPanel

In questa unica pagina si ha un riassunto dei saldi principali dell'articolo in considerazione del documento che si sta compilando. Questa modalità di visualizzazione permette di controllare in modo immediato la giacenza oppure la disponibilità dell'articolo direttamente dal documento prima di emetterlo.

I valori del pannello Dati Articolo vengono visualizzati nel momento in cui si è posizionati in una delle righe di dettaglio e si riferiscono all'articolo presente nella riga e non all'intero documento.
Spostandosi riga dopo riga nella griglia, i valori, presenti nel pannello, relativi agli articoli vengono aggiornati per facilitarne la lettura.

Scheda Dati Principali

Generali

Questa sezione riporta i dati principali dell'articolo al livello aziendale, ovvero senza distinguere il deposito.

Nella sezione Generali sono visualizzate e aggiornate le informazioni relative alle quantità presenti, impegnate e ordinate lette dall'Anagrafica dell'Articolo. I saldi dell'articolo vengono letti dalla scheda Valorizzazione, mentre gli altri dati, come per esempio la Scorta Minima, vengono riportati in base a quanto impostato nella scheda Dati Principali. Per dettagli, si veda: Anagrafica Articolo

Deposito

La sezione presenta gli stessi dati di quantità contenuti nella scheda Generali ma riferiti al Deposito, al Tipo Specificatore e allo Specificatore selezionato.
Di default, nel campo Deposito, viene proposto il deposito indicato nel documento.
Quando nel documento il Tipo Specificatore è su Ignora, sono presentate le quantità del deposito, indipendentemente dal Tipo Specificatore.
Nel caso in cui nel documento il Tipo Specificatore non sia su Ignora, vengono riportate le quantità del deposito relative al Tipo Specificatore indicato nel documento e allo Specificatore se presente.

Prospetto Disp.

Il prospetto disponibilità viene reso disponibile anche in assenza dei moduli di produzione, in quanto evidenzia la disponibilità teorica nel tempo, considerando ordini di acquisto e di vendita, oltre ai versamenti e prelievi di produzione.

Lifo-Fifo

Visualizza i dati dei carichi movimentati tramite il documento se l'articolo è valorizzato secondo la metodologia FIFO/LIFO a scatti continui.

Questa scheda viene visualizzata nei documenti (ad eccezione di quelli di WMS) e non nelle procedure.

Scheda WMS

In questa scheda è possibile visualizzare i dati stock relativamente ai soli depositi attivi per la gestione WMS. In particolare il programma mostra le quantità disponibili nelle diverse Zone ed Ubicazioni. Nella scheda Cross Docking è possibile visualizzare l'elenco di richieste da Cross Docking.
Questa scheda viene visualizzata Solo in presenza del modulo Wharehouse Management System.

Cross References


Contenuto incluso da: Ref-Guide-M4-Sales-Common-CrossReferences

The Cross References lateral panel lists all documents linked to the current document.

Origin Reference: are the documents from which the current document was generated. For example, in the Sales Invoice for Advance, the Customer Order is the source document.

Destination Reference: are the documents that have been generated by the current document. For example, in the Sales Invoice for Advance, the generated inventory entry is the destination document.

Document Line Reference: are the rows of the document that have been generated, which in this case contain the information of the Service.

All documents, both origin and destination references, can be consulted directly by using double click of the mouse.

The explosion level is indicated on the lower bar of the panel: the default value is 1. By increasing the level, you can view the documents that have been originated by the source document, and the documents that have been generated by or by destination documents.

For example, if a sales invoice has generated a ledger, which has generated a receivable, you'll see only the ledger in the destination references, but if you increase the explosion level you will also see the receivable.

Similarly, if in the ledger, the level 1 is displayed, you will see only the ledger of the issued document among the origin references, but if you increase the explosion level you will also see the sales invoice.
